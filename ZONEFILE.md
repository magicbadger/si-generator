## Purpose

A DNS Zone file allows the broadcast parameters to be used to discover the eventual SPI service using a RadioDNS Lookup (as specified in ETSI TS 102 370).

There is a header indicating the system that exported the file, and date.

Each service will have its own section, with a comment header giving the station name, and a new line between each service.

The broadcast bearers (FM, DAB) should be transformed into their domain-name equivalents, and each given a CNAME record pointing towards a placeholder destination

The user should then customise this for their own purposes

## Example

```
; GENERATED BY SI-GENERATOR (https://github.com/magicbadger/si-generator)

; Service Provider  : rbb
; Originator        : RBB
; Created           : 11-03-2026T08:09+01:00

; Station: Antenne CB
0.d531.1148.de0.dab                     	IN	CNAME	antennecb.derbb.radio.ebu.io
0.d531.1148.de.dab                      	IN	CNAME	antennecb.derbb.radio.ebu.io
*.d531.de0.fm                           	IN	CNAME	antennecb.derbb.radio.ebu.io
*.d531.de.fm                            	IN	CNAME	antennecb.derbb.radio.ebu.io

; Station: Antenne FFO
*.d631.de0.fm                           	IN	CNAME	antenneffo.derbb.radio.ebu.io
*.d631.de.fm                            	IN	CNAME	antenneffo.derbb.radio.ebu.io
0.d631.1148.de0.dab                     	IN	CNAME	antenneffo.derbb.radio.ebu.io
0.d631.1148.de.dab                      	IN	CNAME	antenneffo.derbb.radio.ebu.io
```
